## 原文

KV cache is quietly becoming the new memory wall of LLM inference. Long-context workloads do not just need more FLOPS; they need a serving system that can keep reusable attention state close enough to the GPU without burning scarce HBM on cold tokens.

The clean mental model is a tiered KV hierarchy: HBM for hot decode blocks, accelerator-domain fabrics such as UALink/NVLink for latency-sensitive shared memory, CXL/DDR for warm capacity and prefix-cache pooling, and SSD or remote memory for cold context. CXL looks like the nearer-term deployment path because it extends system memory capacity; UALink-like scale-up fabrics are the bet on direct accelerator memory semantics.

This matters for optical computing and silicon photonics because the same AI infrastructure pressure that creates KV pooling also drives demand for lower-latency, higher-radix, lower-power interconnects. If memory state becomes pooled across GPUs, racks, and eventually pods, optical I/O and optical switching stop being “network upgrades” and become part of the memory architecture.

The winning product is not a magic remote-memory box replacing HBM. It is hardware fabric plus placement runtime plus vLLM/SGLang/LMCache-class software, measured by P99 token latency, small-granularity random access, cache-hit behavior, and production RAS.
