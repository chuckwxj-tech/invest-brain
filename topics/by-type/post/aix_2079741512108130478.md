# The memory shortage got me digging into what could replace or supplement today's server memory. There are eight different server memory technologies and architectures that could...

## 原文

The memory shortage got me digging into what could replace or supplement today's server memory.

There are eight different server memory technologies and architectures that could hit the market.

Here's are the ones I like best.

THE REALISTIC ONES FIRST

HBM4E AND CUSTOM HBM

2027 to 2028

Massive bandwidth, tighter integration with AI accelerators.

Big plus.

Downside is it's expensive to package, hard to cool, and packaging capacity is limited.

I think this will be the the most used memory for cutting edge AI systems.

Easy prediction lol.

SOCAMM AND LPDDR6 SOCAMM

2026 to 2028

SOCAMM2 is already entering production, which makes this feel much more real than people realize.

LPDDR6 versions are likely around 2027 to 2028.

More capacity in less space, uses much lower amounts of power.

My concern is the proprietary designs.

That could lock you into their approved platforms.

Less space and less power draw helps this become a major type of memory for dense AI servers.

CXL POOLED MEMORY

2026 to 2029

This lets servers share memory instead of loading every system with expensive DRAM.

Big plus for utilization.

But latency is higher and the software complexity is real.

I don't think this is about speed.

It's about capacity and efficiency.

I think this could surprise people and reduce need for NAND more than is being discussed currently.

MRDIMM SUCCESSORS

2027 to 2029

Higher bandwidth while keeping the familiar DIMM architecture, that makes it easier to install or even update.

The buffer logic adds power, cost and latency though.

I see this primarily as a bridge between DDR5 and DDR6.

DDR6

2029 to 2030

Faster, replaceable, broad industry support.

Still way below HBM bandwidth and scaling is getting harder.

I think DDR6 becomes the standard CPU memory.

Not because it's exciting, because it's the obvious upgrade path.

NOW THE WILD CARDS

ZAM

2029 to 2031

This is the one I'm most excited about honestly.

Could offer HBM-class bandwidth with more capacity, lower power and lower cost.

But bonding yield and cooling are still up in the air.

If it works, this is the biggest wild card on the list.

If it works, this changes the entire conversation around AI memory.

TRUE 3D DRAM

Around 2030 or later

Vertical cell stacking could mean way higher density.

Manufacturing is difficult and yields are uncertain.

I like the potential but I don't its going to be rolled out by 2030.

MRAM, ReRAM AND PHASE-CHANGE MEMORY

2027 to 2030

Nonvolatile memory that could fill the gap between DRAM and SSDs.

Endurance, speed, density and cost are all still tradeoffs.

My take is these become specialized memory tiers, not DRAM replacements.

BOTTOM LINE

There is no single winner here.

HBM or ZAM for bandwidth.

SOCAMM or DDR6 for CPU memory.

CXL for pooled capacity.

Persistent memory for the colder data.

That's my opinion, did I miss one you think will be big?

Or did i discount it?

Teach me i am ready to learn :)
