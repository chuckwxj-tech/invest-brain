三大存储原厂收缩自研 CXL，真正受益的是谁？

@jukan05 提到，据韩国媒体报道，三星电子、SK 海力士和美光已经缩减或停止自研 CXL Controller 的商业化计划。

这说明 CXL 的产业分工正在从存储原厂垂直整合，转向 Controller、DRAM 和整机系统分别由专业厂商负责。CXL Controller 和系统集成的利润池，可能进一步向 $ALAB 、 $MRVL 和 $PENG 迁移。

一、为什么三大存储原厂不愿继续自研 CXL Controller

三大原厂最初希望复制 HBM 的商业模式：把自研 Controller 与 DRAM 封装成完整的高毛利 CXL 模块，再以系统级产品出售。

但 Hyperscaler 想要的不是一个封闭的一体化产品，而是更灵活的分离式架构：

选择性能最好的 CXL Controller，再搭配市场上价格更低、供应更充足的通用 DIMM。

这会让存储原厂陷入一个矛盾。如果继续投入大量研发费用生产专有 Controller，却无法与 DRAM 捆绑销售，就很难获得预期中的高利润；如果大规模推广高价 CXL 一体化模块，又可能减少客户对普通 DIMM 的采购，反过来侵蚀自身最重要的现金牛业务。

所以，停止自研 Controller 不代表 CXL 需求消失，而是存储原厂重新回到自己最擅长的 DRAM 制造，把控制器、交换芯片和整机部署交给专业厂商。

二、Astera Labs（Ticker: $ALAB）：最直接的 CXL Controller 映射

一句话购买理由：ALAB 买的是 CXL Controller 与 AI 互连标准化后的份额集中。

Astera Labs 的核心定位，是为 AI 服务器提供 CPU、GPU、内存和网络之间的连接芯片。CXL 对应的产品是 Leo Smart Memory Controller，主要解决内存扩展、共享、池化，以及服务器对外部 DDR 的管理。

ALAB 的优势不只是做一颗 Controller，而是已经建立了覆盖 PCIe Retimer、Fabric Switch 和 CXL Memory Controller 的完整连接平台。Hyperscaler在搭建 AI 服务器时，可以在同一套软件和管理体系内完成高速互连和内存扩展。

如果存储原厂不再强推专有 Controller，ALAB 面临的潜在垂直整合竞争会下降。云厂商也更容易采用“第三方 Controller＋通用 DRAM”的开放架构，Leo 的设计导入空间随之扩大。

但这里最大的预期差仍需验证：**竞争对手减少，不等于 ALAB 的订单自动增加。**真正的右侧信号是更多 Hyperscaler 进入量产、公司开始单独披露 CXL 收入，或者 Leo 对未来收入指引产生实质影响。

三、Marvell（Ticker: $MRVL）：不只做 Controller，而是做机架级内存网络

一句话购买理由：MRVL 的增量之一是 CXL 从单机扩容走向机架级内存池化。

Marvell 的 Structera 产品线覆盖了 CXL 的三个不同层级。

Structera X 负责为单台服务器增加外部内存；Structera A 在内存附近加入计算能力，减少数据来回搬运；Structera S 则进一步连接多台 CPU 和 GPU，把不同设备的内存组合成可以统一调用的资源池。

新一代 Structera S 30260 可连接 16 或 32 个 CPU／GPU，支持最高 48TB 共享内存和 4TB/s 总带宽，计划在 2026 年第三季度送样。

这意味着 MRVL 交易的并不只是“一颗 CXL Controller”。如果 AI 基础设施从单台 GPU 服务器，升级为整个 Rack 内统一调度算力、内存和网络，MRVL 对应的是机架级交换与互连平台。

相比 ALAB，MRVL 的 CXL 曝光没有那么纯，但路径更完整。它同时参与定制 ASIC、光 DSP、数据中心网络和 CXL，因此可以从整个 AI 数据流架构中获得收入。

风险也在这里：CXL 对 MRVL 目前仍只是众多增长方向之一。下一步需要看到产品送样转向量产、云客户认证，以及 Structera 是否真正形成可见收入，而不是停留在技术路线图。

四、Penguin Solutions（Ticker: $PENG）：把 Controller 和 DRAM 变成客户能直接使用的服务器

一句话购买理由：PENG 买的是 CXL 从芯片方案走向整机收入和实际部署。

PENG 的角色与 ALAB、MRVL不同。它不需要自己设计最强的 Controller，而是采购控制器、DRAM和服务器组件，将它们整合为客户能够直接部署的 AI 基础设施。

其 MemoryAI CXL KV Cache Server 配置了 3TB 本地 DDR5，并可增加最多 8 张 1TB CXL 扩展卡，总内存容量最高达到 11TB。

这套系统主要解决 AI 推理中的 KV Cache 问题。模型上下文越长、并发用户越多，需要保留的中间数据就越大。如果全部放在昂贵且容量有限的 HBM 中，成本会快速上升；如果频繁卸载到 SSD，延迟又会明显增加。

PENG 的方案是在 HBM 和 SSD 之间增加一层大容量 CXL 内存，让更多推理数据留在速度相对较快的 DRAM 中，减少 GPU 因等待数据而空转。

因此，存储原厂不再发展完整 CXL 系统，对 PENG 的意义可能比对单纯 DRAM 厂商更直接：它减少了潜在的一体化整机竞争，也让 PENG 可以继续在不同 Controller 和 DRAM 之间选择最优组合。

不过，PENG 当前增长仍不能全部归因于 CXL。真正需要验证的是 MemoryAI 是否获得更多客户、订单是否可以持续复制，以及整机集成能否带来高于普通硬件业务的利润率。

五、这条产业变化真正意味着什么

如果报道最终得到确认，CXL 的产业链可能形成更明确的分工：
三星、SK 海力士和美光继续生产 DRAM；
$ALAB 提供专业 CXL Controller；
$MRVL 提供 Controller、交换和机架级内存互连；
$PENG 负责模块、服务器与客户部署。

所以，这不是“CXL 失败”，而是原厂试图复制 HBM 的垂直整合模式受阻后，产业重新回到更有效率的专业分工。
三家公司对应的投资逻辑也完全不同：
$ALAB 是最纯的 Controller 份额逻辑；
$MRVL 是更完整的机架级互连平台；
$PENG 是距离真实客户部署和收入最近的系统集成商。

下一步真正需要跟踪的，不是社交媒体上谁被定义为受益者，而是三项实际数据：Hyperscaler客户认证、CXL相关订单，以及这些产品何时开始影响收入和利润率。
