## 原文

My bet: the next major bottleneck in AI scale-up systems will not be compute—it will be memory.

Today, many scale-up architectures are essentially built around Broadcom switches, Ethernet/IP, and proprietary interconnect IP. The network roadmap is becoming clearer, but there is still no equally compelling solution for large-scale, composable memory pooling.

RDMA can extend memory across servers, but should RDMA really be the primary interface for latency-sensitive LLM state?

LLM inference must store and continuously access massive amounts of model state—especially KV cache and increasingly long-term or conditional memory. Keeping all of it in expensive accelerator-local HBM is neither scalable nor economical.

I believe CXL memory pooling can fill this gap.

The architecture I envision is:

A shared memory pool that internally uses CXL memory semantics
Multiple FPGAs or custom ASICs connecting the pool to accelerator fabrics
Protocol-flexible external interfaces, including ESUN, SUE, UALink, or other scale-up fabrics
A target bandwidth on the order of 200 GB/s per node

In other words, the external transport can evolve, but the memory pool itself remains based on CXL.

This creates a clean separation:

CXL provides the memory semantics.
ESUN, SUE, UALink, or future fabrics provide the reach.
Alibaba’s recent scale-up architecture points in a similar direction: tightly coupled accelerator fabrics built around native memory semantics rather than treating every remote access as conventional networking.

The industry has spent enormous effort building bigger accelerator fabrics. The next opportunity is to build a truly open, composable, high-bandwidth memory fabric behind them.

Scale-up compute needs scale-up memory.
