## 原文

内存是一堵墙，大家都想打破它。

内存即服务（Memory as a Service）可能是关键方向。截止今天，这还是早期商业构想，但 $MRVL 的下一代内存架构、数据中心架构不谋而合，触发了我的投资雷达

“内存即服务”里的 HBM、DRAM、SSD 也仍是实体芯片，只是客户可能不再一次性买断，而是租用容量、带宽或整套 AI 内存系统”

1/ 啥意思
这个概念是由 SK 海力士董事长提出的商业构想：客户不直接购买芯片，而是租用使用权

早上服务器 A 需要 2TB，就多分给 A；下午 B 的任务变大，再把一部分容量分给 B。不至于空转，提高利用率，配合“硬件池化 + 软件调度”收费

2/ 现在内存的瓶颈是什么？
现在存储和 GPU 是要焊在一起的。为什么不能把 HBM 放远处租？物理定律不答应：距离 = 延迟

HBM 靠上千根极短连线贴着 GPU 才有那个带宽。

想象一个场景：内存是一个巨大的水池，靠光互联一样，整个池子都连接在所有的 GPU 上，GPU按需拿取，光速互联

3/ “内存即服务”最可能怎么落地？
① 金融化卖法：芯片不变，合同变。按月按 TB 收费，海力士 $MU 等存储厂保留所有权

② 随云出租：海力士不直接面对客户，和 $NVDA 云合作，藏在"内存优化实例"背后收钱

③ CXL 内存池：数据中心里架一个共享大池子，容量动态分配，按 GB·小时计费 （其余技术路线，随时调研后再分享给大家）

④ 推理内存工厂（最终）：HBM+DDR+CXL+SSD+调度软件整包运营，按 token 收费用 token 更便宜

4/
$MRVL 为什么在我雷达上：

它不做内存，做的是池子的"管道和阀门"——CXL 控制器、交换芯片（收购 XConn）、光 DSP、CPO。
无论最后池子是谁的、GPU 是谁的，数据都得走它的路。卖水人逻辑

一切还是构想，海力士自己都说方案未定。方向真实，落地尚早，这是我最近最关注的一条主线之一
